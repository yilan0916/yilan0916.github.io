Java集合框架

在正常实现方法或者非常关注性能时，选择不同的数据结构会带来很大的差异。
这里与大学计算机课程中的数据结构不同，将跳过理论部分，仅介绍如何使用标准库中的集合类。

接口与实现分离

  • 好处：如果有修改需求，可以很轻松地切换到另一种实现中
  • 使用技巧：接口类型存放集合引用，构造集合对象时使用具体的类
  • 例如：List<User> list = new ArrayList<>();
  • 拓展：研究API文档时，会发现以Abstract开头的类，如AbstractQueue，这些是为类库是闲着设计的。如果想要实现自己的队列可以拓展这些抽象类。

Collection接口

在java类库中，集合类的基本接口是Collection接口

迭代器

使用方法

注意：Java迭代器的查找操作和位置变更紧密耦合，查找一个元素的唯一方法是调用next。在执行查找操作的同时，迭代器的位置就会随之向前移动。因此，可以认为Java迭代器位于两个元素之间，当调用next时，迭代器就会越过下一个元素，并返回刚刚越过的那个元素的引用。（Iterator.next与InputStream.read可以看成等效的，读取一个字节就自动消耗掉一个字节）

remove和next是存在依赖性的，remove只能删除next跳过的元素且不能连续执行两次remove

泛型实用方法

由于Collection和Iterator都是泛型接口，里面有很多很有用的方法。

集合框架中的接口

具体集合

集合类型 描述
ArrayList 可以动态增长和缩减的一个索引序列
LinkedList 可以在任何位置高效插入和删除的一个有序序列
ArrayDeque 实现为循环数组的一个双端队列
HashSet 没有重复元素的一个无需集合
TreeSet 一个有序集
EnumSet 一个包含枚举类型值得集
LinkedHashSet 一个可以记住元素插入次序的集
PriorityQueue 允许高校删除最小元素的一个集合
HashMap 存储键/值关联的一个数据结构
TreeMap 键有序的一个映射
EnumMap 键属于枚举类型的一个映射
LinkedHashMap 可以记住键/值项添加次序的一个映射
WeakHashMap 值不会再别出使用时就可以被垃圾回收的一个映射
IdentityHashMap 用==而不是equal比较键的一个映射

链表

LinkedList，如果同时使用两个迭代器进行修改，会爆异常。
在使用时，要注意使用多个迭代器的前提是这些迭代器只能读取或者再关联一个能同时读写的迭代器。

并发修改有一个奇怪的例外，链表只跟踪列表的结构性修改（如添加和删除），set方法不视为结构性修改

尽量不使用LinkedList的get方法，这是一种虚假的随机访问，实际上没有缓存位置信息、效率低下。

数组列表

ArrayList与Vector的区别

ArrayList封装了一个动态再分配的对象数组，支持随机访问。
Vector是动态数组，它的所有方法都是同步的，因此可以安全地从两个线程访问一个Vector对象。代码会在同步上花费大量的时间。
因此：在不需要同步的时候使用ArrayList，需要同步的时候使用Vector

散列集

散列集能够快速查找元素的数据结构，缺点是无法控制元素出现的次序。

如果是自己定义的类需要负责实现hashCode和equals方法，若a.equals(b)为true，则ab必须有相同的散列码。

在Java中，散列集是用链表数组实现的，每个列表称为桶(bucket)。若未出现的对象发生了散列冲突，则添加在链表后面。填装因子通常设置为0.75，（在Java8中，桶满时会从链表变为平衡二叉树）

HashSet实现了基于散列表的集。

注意：更改集中的元素要格外小心，如果元素的散列表发生了改变，元素中数据结构的位置也会发生 改变

树集

树集TreeSet和散列集十分类似，不过它比散列集有所改进。
树集时一个有序集合，它是通过红黑树实现的。每次添加元素时都能将其放在正确的位置上。

特点：添加时添加元素时比散列表慢，查找元素时很快（时间复杂度为logn）

要使用树集，必须能够比较元素，这些元素必须实现Comparable接口，或者中造集时提供Comparator方法。

队列与双端队列

双端队列（deque），在ArrayDeque和LinkedList中都实现了双端队列。
在12章中会看到限定队列和限定双端队列。

优先队列

优先队列使用的数据结构是堆，也同样没有对元素进行排序。
与TreeSet一样，对象实现了Comparable接口，或者构造器提供Comparator方法。

典型用法是任务调度

映射

映射数据结构就是map，用来存放键值对的，如key为id，value为User

基本映射操作

Java类库提供了两种通用的实现：HashMap（散列映射）和TreeMap（树映射）。

与集一样，散列映射比树映射快一点，如果不需要按照顺序访问键，最好选散列映射

当key不存在时，调用get方法为返回null。若null不方便，可以使用默认值
map.getOrDefault(id, 0)

更新映射条目

更新的时候必须考虑一个特殊情况，那就是键第一次出现。例如，统计一个单词（word）的出现的次数

映射视图

集合框架不认为映射本身是一个集合，但它的视图是集合。
映射有3种视图：键集、值集合（不是一个集）、键值对集

注意：keySet既不是HashSet，也不是TreeSet，是实现Set接口的另外某个对象

键集视图可以调用迭代器的remove，实际上会从映射中删除这个键和它关联的值。但是==不能添加元素==

java10中引入推断类型：var

弱散列映射

出现的原因：当对某个键的最后一个引用消失，那么不再有任何途径可以引用这个值的对象。遗憾的是，垃圾回收器不能删除它们。

垃圾回收器会跟踪活动的对象。只要映射对象是活动的，其中的所有桶也是活动的，它们不能被回收。因此只能程序从映射表中删除或者使用WeakHashMap

WeakHashMap使用弱引用（weak reference）。WeakReference包含散列表键，会周期性的检查队列以便找到新添加的弱引用，弱引用一旦进入队列意味着不可再被使用并且已经回收。

链接散列集与映射

LinkedHashSet和LinkedHashMap是有序的，可以按插入顺序（默认）、也可以按访问顺序。

按访问顺序是值调用get或put时，受影响的项将从当前位置移动到链表的尾部，这对缓存实现“最近最少使用”原则十分重要

甚至可以实现自动化，自动删除最近最少使用的项

枚举集与映射

EnumSet是枚举类型元素集的搞笑实现，内部用位序列实现。

EnumSet没有公共的构造器，需要使用静态工厂构造这个集

EnumMap是一种键类型为枚举的映射，可以高效的实现值数组

标识散列映射

IdentityHashMap有特殊的用途。在这个类中，键的散列值不是用hashCode计算的，而是用System.identityHashCode计算的。这个Object.hashCode根据对象的内存地址计算散列所用到的方法。而且，在对两个对象进行比较时，IdentityHashMao使用==，而不是equals。

也就是说，不同的键对象即使内容相同，也被视为不同的对象。在实现对象遍历算法（如对象串行化）时，这个类非常有用，可以用来跟踪哪些对象已经遍历过

视图与包装器

可以使用视图（view）来获取其他实现了Collection接口或者Map接口的对象。

如前面提到的keySet方法，初看起来，好像是这个方法创建了一个新集，并填入映射的所有键，然后返回这个集。但是，情况并非如此。事实上，keySet方法返回一个实现Set接口的类对象，有这个类对象的方法操纵原映射。这种集合称为视图。

关于可选操作的说明

在集合和迭代器的接口的API文档中，许多方法描述为“可选操作”。

是否应该将可选方法这一技术拓展到自己的设计中呢？不应该！
在自己的变成问题中，很少遇到这种极端的约束，你应该能够找到一种合适的解决方案，而不必依赖“可选”操作这种极端做法。

算法

为什么使用泛型算法

泛型集合有一个很大的优点，即算法只需要实现一次。
请考虑为了高效得执行算法所需要的最小集合接口。如计算最大元素，这项任务不需要随机访问 因此可以将max方法实现为能接收任何实现了Collection接口的对象。

Java类库中有排序、二分查找和一些实用算法

排序和混排

排序

java的排序是将所有元素转入一个数组，对数组进行排序，然后将排序后的序列复制回列表。
集合类库中排序不是快排（快排不稳定），采用的是归并排序，它是稳定的，时间复杂度nlogn。
